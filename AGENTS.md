# Codex Project Instructions

このリポジトリは `Projekt Black` のGodot 4.6.3プロジェクトです。Codexは以下の方針を守って作業します。

## Communication

- ユーザーには日本語で説明する。
- 専門用語を使う場合は、初心者にも分かる短い説明を添える。
- 仕様が曖昧な場合は、危険な推測で実装せず、仮定を明記する。
- 大きな変更の前には、何を変更するかを簡潔に説明する。

## Technical Baseline

- Godot 4.6.3 stableを前提にする。
- 参照する公式資料はGodot公式ドキュメントを優先する。
- `latest`ドキュメントは将来版の内容を含む可能性があるため、実装前に4.6.3で使えるか確認する。
- 基本言語はGDScriptとする。
- GDScriptは可能な限り型を付ける。
- 2Dゲームを主軸にし、必要な場合だけ3D、パーティクル、シェーダーを使う。

## Safety Rules

- 変更前に関連ファイルを読む。
- 変更後に `git diff` で差分を確認する。
- ひとつのコミットには、ひとつの意味のある変更だけを入れる。
- `main` ブランチは常に起動可能な状態を目指す。
- 破壊的な変更や大規模リファクタの前には、必ず理由と影響範囲を説明する。
- ユーザーが作った未コミット変更を勝手に戻さない。
- 作業終了時は `docs/HANDOFF.md` を更新する。

## Project Structure

- `scenes/`: Godotシーン。画面に置かれるまとまり。
- `scripts/`: GDScriptコード。
- `resources/`: 武器、敵、アイテム、スキルなどのデータ。
- `assets/`: 画像、音声、VFX、フォント。
- `docs/`: 設計、引き継ぎ、制作ルール。
- `tools/`: 補助スクリプトや検証用ツール。

## Content Handling

- ゴア、成人向け、センシティブな素材名は中立的な名前にする。
- コード上では内容を記号的に扱い、露骨な説明を残さない。
- 登場キャラクターは全員18歳以上として扱う。
