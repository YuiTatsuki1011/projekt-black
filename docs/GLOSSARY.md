# Glossary

初心者向けの用語集です。今後の説明では、ここにある意味で使います。

## Godot

ゲームを作るためのソフトです。絵、音、物理、入力、シーン管理などをまとめて扱えます。

## Project

ゲーム全体の作業フォルダです。このプロジェクトでは `D:\projekt-black` が該当します。

## Scene

Godot内の「部品」です。プレイヤー、敵、弾、UI、部屋などをシーンとして作れます。

## Node

シーンの中に置く小さな機能部品です。見た目を表示するNode、当たり判定を持つNode、音を鳴らすNodeなどがあります。

## Script

動きを書くコードです。たとえば「Dキーを押したら右に動く」という処理を書きます。

## Resource

データの入れ物です。武器の攻撃力や弾数など、数値だけを保存したいときに使います。

## Signal

Godotの通知機能です。たとえば「弾が敵に当たった」ときに、別の処理へ知らせるために使います。

## Autoload

ゲーム中ずっと存在する共有スクリプトです。便利ですが、増やしすぎるとコードが複雑になります。

## Commit

Gitで作業状態を保存することです。ゲーム制作で言えば「復元ポイント」に近いものです。

## Branch

Git上の作業場所です。`main` を安全な本線として残し、新機能は別のbranchで作ります。

## Pull Request

GitHub上で、branchの変更をmainへ取り込む前に確認する仕組みです。

## Vertical Slice

完成版の一部だけを小さく作った試作品です。このゲームでは、移動、照準、射撃、敵、死亡までを最初の縦切りにします。

## Placeholder

仮素材です。完成アートが入る前に、四角や簡単な画像で動作確認するために使います。
