# Roadmap

このロードマップは、破綻しにくい順番で作るための実装計画です。

## Phase 0: Foundation

- GitとGitHubの接続
- 制作プロトコル整備
- ディレクトリ構成整備
- 最初の引き継ぎ文書作成

## Phase 1: Core Controls

目的: プレイヤーを操作して、狙って撃てるようにする。

- プレイヤー移動
- ジャンプ
- しゃがみ
- マウス照準
- 銃の向き制御
- 発砲
- 弾薬
- リロード

## Phase 2: Basic Combat

目的: 戦闘の最小単位を作る。

- 弾の当たり判定
- 敵1種類
- HPとダメージ
- 死亡処理
- 血液VFX
- ダメージ反応
- 射線予告つきの敵遠距離攻撃

## Phase 3: Test Level

目的: 小さな部屋でプレイ感を確認する。

- テスト用タイルマップ
- 足場
- 扉
- アイテム配置
- 簡単なUI

## Phase 4: Loot and Items

目的: ルーターシューターの基礎を作る。

- アイテムデータ
- 武器データ
- ドロップ処理
- インベントリの最小実装
- 装備変更

## Phase 5: Progression

目的: RPGとして育成できるようにする。

- 経験値
- レベル
- スキルツリーの試作
- ステータス補正

## Phase 6: Procedural Areas

目的: 侵入ごとに変化するマップの基礎を作る。

- 部屋テンプレート
- ランダム接続
- 敵配置
- アイテム配置
- 脱出地点

`Procedural` とは、手作業で全部置くのではなく、ルールに従ってゲーム側が自動生成することです。

## Phase 7: Quest and Extraction

目的: 依頼を受け、目標を達成し、脱出する流れを作る。

- 依頼データ
- 目標敵
- 達成判定
- エクストラクションポイント
- 報酬
